# Open Questions & Decisions

## Interface & Interaction
- [ ] Is this a web app, desktop app, CLI, or just a markdown folder structure that users edit directly in their text editor?
- [ ] Does it need a UI layer at all, or is it purely about defining the markdown structure + tools to parse/generate it?
- [ ] Should there be read-only views/dashboards, or is editing the primary interface?

## Storage & Sync
- [ ] Where do these markdown files live? Local filesystem, cloud (GitHub, Obsidian Sync, iCloud), self-hosted server?
- [ ] Do they version-control via Git, or is that assumed?
- [ ] Should the system support offline editing with conflict resolution?

## Planning Hierarchy & Relationships
- [ ] How do life goals/values inform annual plans? Are there explicit links between them, or just conceptual?
- [ ] Weekly plans → daily plans: are daily plans *derived* from weekly plans, or can they be independent?
- [ ] When you complete a daily plan, does that feed back up to inform the weekly reflection?
- [ ] Is there an explicit relationship (linking/referencing) between levels, or is it structural (nested folders/files)?

## Daily Reflection & Healthy Behaviors
- [ ] What does "end of day reflection" look like structurally? Free-form notes, structured checklist, both?
- [ ] When you commit to a "healthy behavior" (e.g., "exercise 3x/week"), how is that tracked? Checkbox, counter, duration?
- [ ] Should the system show streaks, compliance %, or just record the raw data?
- [ ] Are healthy behaviors independent from daily plans, or part of them?

## Long-term Projects & Recurring Tasks
- [ ] Do recurring tasks live in a separate "recurring task registry" or are they embedded in weekly/daily plans?
- [ ] For a "house remodel" project spanning 6+ months, does it get its own file, or live in annual/quarterly plans?
- [ ] How are recurring tasks triggered/surfaced? (E.g., "furnace filter every 6 months" — when is the user reminded?)
- [ ] Should completed instances of recurring tasks be archived, or just marked as done in-place?

## Scope & MVP
- [ ] What's the *minimum* that would be useful? (E.g., just life goals + annual + daily, without quarterly/weekly?)
- [ ] Is this single-user or multi-user?
- [ ] What's the primary use case: planning, tracking, reflection, or all three equally?

## User Experience
- [ ] How does a typical user interact with this on a daily basis? (E.g., morning planning, evening reflection, weekly review)
- [ ] Should the system surface suggestions or nudges based on the hierarchy? (E.g., "You haven't touched your Q2 goals this week")
